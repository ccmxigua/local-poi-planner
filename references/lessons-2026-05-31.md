## 1. 性能瓶颈定位走过的弯路（重要）

排查 planner 卡住/慢时，结论被反复推翻，正确顺序是：

1. ❌ 一开始以为是高德 transit API 慢 → 实测 transit 0.2–0.5s、POI 召回 2.3s，都不慢。
2. ❌ 然后以为是 Exa 402 / 超时 → 是表象，不是主因。
3. ❌ 再以为是 Tavily 自建代理 fetch() 无超时挂死（113s）→ 这是在 **legacy 脚本** 上测的，跟实际路径无关。
4. ✅ 真正根因：planner 调的是新 `unified-search.sh`（deep search-layer），且 `build_queries` 会对每个 POI 生成多条 query，planner **串行** 发 7+ 次 unified-search，每次约 76s，累计 >9 分钟，看着像死锁，其实是「调用次数过多 + 单次慢」。

**教训**：profile 一定要 profile 实际跑的那条路径（`run-search-layer.sh` → `search.py`），不要在 legacy 脚本上得结论。先确认入口脚本和分支，再测延迟。

## 2. 搜索质量：只有 Grok 对中文本地 query 可用

- Exa：402（key 余额不足），新脚本对缺引擎无降级。
- Grok：需正确配置才可用；`enable_search:true` 会让 Surge 代理（198.18.2.223）挂死返回 HTTP 000，须 `proxies={"http":None,"https":None}` 绕过；中文多智能体 query 耗时 22–31s，超时须设到 1800s（30 分钟），不是 120s。
- Tavily / TinyFish：对本地中文 query 基本是垃圾（鸦片战争维基、学术 PDF、政府文件、汽车广告），污染约 87% 输出。
- 配置位置：`~/.openclaw/credentials/search.json`；模型 `grok-4.20-multi-agent-xhigh`。

**教训**：本地店铺类中文 query 实际只有 Grok 有效。要降污染应让 deep mode 默认 `--source grok`（注意 `--source` 在 deep 路径目前被忽略，是已知 bug）。

## 3. unified-search 参数死代码

- planner 把 `5 general` 拼进 query 字符串，但 **没有** 作为 `--num` / `--topic` 参数传给 `unified-search.sh`；deep mode（`run_search_layer_auto`）也忽略这些。属死代码，不影响行为。用户已确认「算了，不动」。

## 4. 交通明细渲染 bug（已修，2026-05-31）

改 `amap_direction.py` 的 `get_transit_details` + `format_transit_detail_lines`：

- **同 segment 多条 buslines 是「同一段的备选线路」**（同起终点），旧代码并列渲染成多个「乘坐」，看着像连坐 3 趟车。→ 合并为主线路 + `alternatives`。
- `via_stops` 是含站名的 **数组**（dict.name），旧代码只取了 `via_num` 计数。→ 提取 `via_stop_names`，渲染「途经：…」。
- `walking.steps` 有完整 turn-by-turn，旧代码只取 `steps[0]`。→ 保留全部 `instructions`，逐条列出。
- 地铁线路也走 `bus.buslines`，`type` 含「地铁」。→ 新增 `is_subway`，前缀「乘坐地铁」。
- `max_parts=6` 会截断路段。→ 默认改 `max_parts=0`（不截断）。

调用点不变：`planner.py:531` 调 `get_transit_details`，`:535` 调 `format_transit_detail_lines`。

## 5. 可达性评分与阈值

- transit 评分：`max(0, 100 - duration_min * 2)`，>50 分钟得 0。
- walking 评分：`max(0, 100 - duration_min * 3)`，>33 分钟得 0。
- 总分：`total_score = original_score * 0.7 + accessibility_score * 0.3`。
- 可达性标签（`_accessibility_label`，planner.py）：transit ≤30→✅地铁可达，30–45→⚠️勉强可达，>45→❌公交不便；walking ≤15→✅步行可达，15–25→⚠️步行较远，>25→❌步行不便。
- **现实约束**：旗山校区周边店铺普遍离地铁远，多数得 0，不是代码 bug 是地理现实。用户已确认评分逻辑「都不动」。
